**Gap 3: .toFixed(1) bypassing locale number formatting (FMT-02)**

All five files already import `useFormatter` from 'next-intl' and initialize `const format = useFormatter()`. The fix is a single-expression replacement in each file.

The replacement pattern: change `value.toFixed(1)` to `format.number(value, { maximumFractionDigits: 1, minimumFractionDigits: 1 })`. This ensures a Romanian user sees "7,5" instead of "7.5".

Specific changes:

1. **summary-screen.tsx line 184**: Replace `{sessionScore.toFixed(1)}` with `{format.number(sessionScore, { maximumFractionDigits: 1, minimumFractionDigits: 1 })}`. The `format` variable is already initialized at line 124.

2. **context-panel.tsx line 250**: Replace `stats.avgScore.toFixed(1)` with `format.number(stats.avgScore, { maximumFractionDigits: 1, minimumFractionDigits: 1 })`. There are two `format` initializations in this file (lines 108 and 273) -- use the one in scope for line 250 (line 108, in the ContextPanelDesktop component).

3. **recap-screen.tsx line 87**: Replace `lastSession.sessionScore.toFixed(1)` with `format.number(lastSession.sessionScore, { maximumFractionDigits: 1, minimumFractionDigits: 1 })` inside the `t("score", { score: ... })` call. The `format` variable is at line 46.

4. **floating-context-widgets.tsx line 407**: Replace `stats.avgScore.toFixed(1)` with `format.number(stats.avgScore, { maximumFractionDigits: 1, minimumFractionDigits: 1 })`. The `format` variable is at line 277 (FloatingContextWidgets component scope).
