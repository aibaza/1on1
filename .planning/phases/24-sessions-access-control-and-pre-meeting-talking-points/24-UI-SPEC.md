## Layout Contract

### Sessions Page — Admin View

```
[Page Header: "Sessions" + "New Series" button]

[Manager Name Group Header: "Alex Johnson (You)"]        <- own group first
  [SeriesCard] [SeriesCard] [SeriesCard]                  <- 3-col grid

[Separator: 1px border line, 24px vertical margin]

[Manager Name Group Header: "Diana Martinez"]             <- alphabetical
  [SeriesCard] [SeriesCard]

[Manager Name Group Header: "Sam Wilson"]
  [SeriesCard]

[Archived collapsible]                                    <- existing pattern
```

- Group header: 14px semibold, flush left, 8px margin-bottom before grid
- Groups separated by `<Separator />` (1px border) with 24px vertical spacing
- Own group identified by appending " (You)" in muted-foreground weight 400
- Admin with no direct reports: no "(You)" group, starts with first alphabetical manager

### Sessions Page — Manager View

```
[Page Header: "Sessions" + "New Series" button]

[Section Label: "My Team"]
  [SeriesCard] [SeriesCard] [SeriesCard]                  <- 3-col grid

[Section Label: "My 1:1s"]                                <- 32px top margin
  [SeriesCard with manager name top-right]

[Archived collapsible]
```

- Section labels ("My Team", "My 1:1s"): 14px semibold, 8px margin-bottom
- "My 1:1s" section only shown if the manager also appears as a report in any series
- Cards in "My 1:1s" show the manager's name in top-right: 12px regular, muted-foreground

### Sessions Page — Member View

```
[Page Header: "Sessions"]                                 <- no "New Series" button

[SeriesCard] [SeriesCard]                                 <- flat grid, no sections
```

- No grouping, no section headers
- Standard 3-col responsive grid (1-col mobile, 2-col md, 3-col lg)

### Series Card — Agenda Button Placement

```
[Card]
  [Avatar] [Name + Stars]          [Status Badge] [Agenda btn]
  [Summary blurb]
  [In-progress label]
  [Start/Resume btn]               [Calendar + schedule]
```

- Agenda button: `variant="ghost"` `size="icon"`, positioned in the card header row, after status badge (or in its place if no badge)
- Icon: `ListTodo` from lucide-react, 16x16px (h-4 w-4)
- Count badge: absolute positioned top-right of button (-top-1 -right-1), `h-4 min-w-4 px-0.5 text-[10px]` font-semibold, primary background
- Only shown when `latestSession?.status === "scheduled"`
- `z-10` to sit above the card's full-cover Link overlay
- `onClick` calls `e.preventDefault()` + `e.stopPropagation()` to prevent card navigation

### Agenda Sheet

```
[Sheet: right side, 400px wide on desktop, full-width on mobile]
  [SheetHeader]
    [SheetTitle: "Alex Johnson"]
    [Subtitle: "Session 4 . Mar 20, 2026"]

  [Sheet body: scrollable, py-24px]
    [Category Section: "General"]                         <- Collapsible, open by default
      [TalkingPointList]

    [Category Section: "Wellbeing"]
      [TalkingPointList]

    [Category Section: "Performance"]
      [TalkingPointList]

    --- OR if no categories/points ---

    [EmptyState]
      icon: ListTodo
      heading: "No agenda items yet"
      body: "Add talking points to prepare for your upcoming meeting."
```

- Sheet side: `right`
- Sheet width: `sm:max-w-[400px]` (shadcn default side sheet behavior, full-width on mobile)
- Category sections use `Collapsible` with `defaultOpen={true}`
- Category header: 12px medium text, uppercase tracking-wide, muted-foreground (matches wizard context panel pattern)
- Vertical spacing between categories: 16px
- `readOnly={false}` for discussed toggle -- users can mark items as informally discussed pre-meeting
- Sheet body has `overflow-y-auto` for scrolling when many talking points exist

## Interaction Contract

### Agenda Button

| Interaction | Behavior |
|-------------|----------|
| Click | Opens agenda sheet (right slide-in) |
| Hover | Ghost button hover state (foreground icon) |
| Visibility | Only when series has a scheduled (not started) session |

### Agenda Sheet

| Interaction | Behavior |
|-------------|----------|
| Open | Slide in from right, overlay dims background |
| Close | Click X, click overlay, or press Escape |
| Add talking point | Type in input field, press Enter or click add button |
| Toggle discussed | Click checkbox on talking point row |
| Delete talking point | Click trash icon on talking point row |
| All mutations | Optimistic updates via TanStack Query (existing TalkingPointList behavior) |

### Role-Based Grouping

| Interaction | Behavior |
|-------------|----------|
| Admin page load | Series grouped by manager, own group first, rest alphabetical |
| Manager page load | Series split into "My Team" and "My 1:1s" sections |
| Member page load | Flat list of own series only |
| TanStack Query refetch | API route returns role-filtered data; no flash of unfiltered content |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Agenda button | `aria-label` with person name + "agenda" (e.g., "Alex Johnson agenda") |
| Sheet | Focus trap when open (handled by Radix Dialog/Sheet primitive) |
| Sheet close | Escape key closes sheet (handled by Radix) |
| Section headers | `<h3>` for manager group names (admin view) and section labels (manager view) |
| Talking points | Existing TalkingPointList a11y maintained (checkbox labels, delete button labels) |
| Count badge | `aria-hidden="true"` -- the count is supplementary to the button label |
