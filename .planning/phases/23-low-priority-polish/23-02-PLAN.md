**Step 2: Add the link in series-card.tsx.**
Find the no-summary placeholder block (around lines 477-487) where `summaryPlaceholder` is rendered when `!series.latestSummary`. After the placeholder `<p>` tag, add a conditional link that appears ONLY when:
1. `isManager` is true (variable already computed in the component)
2. `!series.latestSession` (no sessions exist yet)

The link should be a `<button>` element that calls `startSession.mutate()` (the mutation is already defined in the component). Follow the existing click propagation pattern from the Start/Resume button:

```tsx
{isManager && !series.latestSession && (
  <button
    onClick={(e) => {
      e.preventDefault();
      e.stopPropagation();
      startSession.mutate();
    }}
    className="relative z-10 pointer-events-auto text-xs text-primary hover:underline mt-1"
  >
    {t("series.startFirst")} {"\u2192"}
  </button>
)}
```

Key details:
- `relative z-10 pointer-events-auto` is required to escape the full-card Link overlay (existing pattern in this file)
- `e.preventDefault(); e.stopPropagation()` prevents the card link from firing
- `text-xs text-primary hover:underline` matches the UI-SPEC interaction contract
- `mt-1` provides a small gap below the placeholder text
- `\u2192` is the right arrow character (arrow)
- The `startSession.mutate()` call is already defined in the component — it creates a new session and navigates to the wizard
